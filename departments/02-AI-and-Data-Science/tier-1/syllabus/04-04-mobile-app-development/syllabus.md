## Track Overview

**Who this is for:** Aspiring mobile developers targeting iOS/Android cross-platform roles.

**Prerequisites:** Basic programming helpful; JavaScript (React Native) or Dart (Flutter) taught from fundamentals.

**Framework choice:**
| Framework | Language | Best For |
|-----------|----------|----------|
| **Flutter** | Dart | UI-rich apps, Google ecosystem, single codebase |
| **React Native** | JavaScript/TypeScript | Web dev crossover, large npm ecosystem |

**Tools:** Android Studio, Xcode (Mac for iOS), VS Code, Flutter SDK / React Native CLI, Expo (RN), Firebase, SQLite/Hive.

**Certification targets (9-month):** Play Store / App Store published app

## 2-Week Crash Course (Pick Flutter OR React Native)

**Mini Project:** Deploy mini app | **Level:** L1

| Days | Topics | App Built | Self-Check |
|------|--------|-----------|------------|
| D1–2 | Setup, widgets/components, layouts | Hello World + UI screens | Explain widget/component lifecycle |
| D3–4 | Navigation, forms, state basics | Calculator app | Stateful vs stateless difference |
| D5–6 | API integration (REST), local storage | Weather / News app | async/await or Future handling |
| D7–8 | Authentication flow, push notifications intro | Login + profile app | Secure token storage approach |
| D9–10 | Build + APK/IPA basics | **Deploy mini app** | Walk through build pipeline |

**Flutter vs React Native parallel labs:** Same app specs; framework-specific syntax handouts provided.

## Appendix: Mentor Notes

**Common pitfalls:** Not testing on physical devices; hardcoded API URLs; ignoring platform-specific UI guidelines; large image assets causing slow loads.

**Hardware:** Android device or emulator required; Mac needed for iOS builds in production track.

**Cross-track:** [Track 3 — Dart/JavaScript](03-programming-languages.md); [Track 1 — Full Stack](01-full-stack-development.md) for API backend.
