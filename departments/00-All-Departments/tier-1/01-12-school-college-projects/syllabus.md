## Service Overview

**Who this is for:** School students (mini projects), college students (major/final-year projects), and students needing guided implementation alongside academic requirements.

**Prerequisites:** Enrollment in or completion of relevant technical track topics; project topic approved by mentor.

**Support model:** Milestone-based deliverables with mentor touchpoints — not daily coding labs.

**Project types:** Mini (IoT simulation, web app) → Academic (ML-based, Android app) → Major (full-stack + cloud + AI integration)

| Duration | Support Level | Key Deliverables |

## 2 Weeks — Topic & Synopsis Support

**Deliverables:** Topic selection + synopsis + basic implementation guidance

| Milestone | Day | Deliverable | Mentor Review |
|-----------|-----|-------------|---------------|
| M1 | D1–2 | 3 topic options with feasibility analysis | Select 1 topic |
| M2 | D3–5 | Synopsis (2 pages): problem, objective, methodology | Approve synopsis |
| M3 | D6–10 | Basic implementation plan + tool list | Q&A session |

**Acceptance criteria:**
- [ ] Topic aligned with academic requirements and institution guidelines
- [ ] Synopsis includes problem statement, objectives, scope, and tools
- [ ] Implementation plan with week-by-week student tasks

---

<div class="duration-section"></div>

## 1 Month — Mini Project Support

**Deliverable:** Full mini project + documentation template

| Week | Milestone | Student Output |
|------|-----------|----------------|
| W1 | Requirements + design | SRS (5 pages) + wireframes/flowchart |
| W2 | Core implementation | Working prototype (50% features) |
| W3 | Complete implementation + testing | Full working project |
| W4 | Documentation + presentation | Report draft + PPT (10 slides) |

**Acceptance criteria:**
- [ ] Project runs and demonstrates core functionality
- [ ] Documentation follows institution template
- [ ] Screenshots and test cases included

---

<div class="duration-section"></div>

## 3 Months — Major Project Module Development

**Deliverable:** Major project modules with weekly mentor reviews

| Month | Focus | Milestones |
|-------|-------|------------|
| M1 | Requirements, design, architecture | SRS, UML diagrams, DB schema |
| M2 | Module 1 + Module 2 implementation | Frontend + backend core |
| M3 | Integration, testing, documentation | Integrated system + test report |

**Weekly mentor review covers:** Progress vs plan, code quality, blockers, next-week goals.

**Acceptance criteria:**
- [ ] All modules integrated and tested
- [ ] ER diagram, DFD, UML included
- [ ] Unit test results documented

---

<div class="duration-section"></div>

## 6 Months — Complete Final-Year Project

**Deliverable:** Full final-year project (frontend + backend + DB) + testing

| Phase | Weeks | Focus |
|-------|-------|-------|
| Planning | 1–4 | Requirements, design, tech stack finalization |
| Development | 5–16 | Iterative module development (2-week sprints) |
| Testing | 17–20 | Unit, integration, UAT |
| Documentation | 21–24 | Final report, slides, demo video |

**Acceptance criteria (L4):**
- [ ] Full-stack or equivalent major project deployed or demo-ready
- [ ] Test cases with pass/fail results
- [ ] Final report 40+ pages (institution format)
- [ ] 15-minute demo video

## 9 Months — Research-Oriented Project

**Deliverable:** Research project + paper writing + viva prep

| Phase | Focus |
|-------|-------|
| M1–M2 | Literature survey, problem definition, methodology |
| M3–M6 | Implementation + experimentation + results |
| M7–M8 | Paper writing (IEEE/APA format) |
| M9 | Viva preparation + defense coaching |

**Acceptance criteria (L5):**
- [ ] Literature review (20+ references)
- [ ] Experimental results with analysis
- [ ] Draft paper submitted to mentor
- [ ] 50+ viva Q&A practiced

## Appendix: Mentor Notes

**Common pitfalls:** Topic too broad; starting coding before design; ignoring college formatting guidelines; plagiarism in report.

**Policy:** All code must be student-authored with mentor guidance; plagiarism check required before final submission (see [Track 13](13-project-report-thesis-writing.md)).

**Cross-track:** Report writing support in [Track 13](13-project-report-thesis-writing.md); viva prep in [Track 14](14-career-development-soft-skills.md).
