# Track 28: QA Automation

**MR ROKESH TECHNOLOGY — Training & Internship Programs**

</div>

> **Skill path:** Manual Testing → Test Design → Selenium → API Testing → CI/CD → Performance → Framework Design

**Related:** [Track 1 — Full Stack](01-full-stack-development.md) | [Track 15 — DevOps Engineering](15-devops-engineering.md)

---

## Track Overview

**Who this is for:** Aspiring QA engineers, SDETs, and developers adding test automation skills.

**Prerequisites:** Basic understanding of web applications; programming (Java/Python/JavaScript) for automation tiers.

**Tools:** Selenium WebDriver, Playwright, Cypress, Postman/Newman, JUnit/TestNG/Pytest, Jenkins/GitHub Actions, JMeter/k6.

**Certification targets:** ISTQB Foundation (optional, 6-month)

## 2-Week Crash Course

| Days | Topics | Deliverable |
|------|--------|-------------|
| D1–2 | SDLC, test levels, test case design | 20 test cases for sample app |
| D3–4 | Selenium WebDriver basics (Java/Python) | 5 automated UI tests |
| D5–6 | XPath/CSS selectors, waits, Page Object Model | POM-based test class |
| D7–8 | API testing with Postman | 10 API test cases + collection |
| D9–10 | Test reporting, capstone | **Smoke test suite + test report** |
