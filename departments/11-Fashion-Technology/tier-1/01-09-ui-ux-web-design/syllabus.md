## Track Overview

**Who this is for:** Aspiring UI/UX designers, product designers, and front-end developers with design focus.

**Prerequisites:** None; creativity and attention to detail emphasized.

**Tools & lab environment:** Figma (free tier), FigJam, Maze/UsabilityHub (testing), Notion/Google Docs (research), HTML/CSS/Tailwind (3-month+ implementation labs).

**Certification targets:** Portfolio-based; Google UX Design Certificate alignment (optional)

## 2-Week Crash Course

**Mini Project:** Complete Figma prototype | **Level:** L1

| Days | Topics | Deliverable | Self-Check |
|------|--------|-------------|------------|
| D1–2 | UX vs UI, design principles, color/typography | Mood board | Explain visual hierarchy |
| D3–4 | Figma basics: frames, components, auto-layout | Mobile app UI screens (3) | Component vs instance |
| D5–6 | Wireframing, user flows, low-fi prototypes | App wireframe set | User flow vs user journey |
| D7–8 | High-fidelity design, responsive layouts | Landing page design | 8pt grid system |
| D9–10 | Usability testing basics, developer handoff | **Complete Figma prototype** | Export assets for dev |

**Submission checklist:**
- [ ] Figma file with organized pages and components
- [ ] Interactive prototype with 3+ linked flows
- [ ] Design handoff notes (spacing, colors, fonts)

## 1-Month Foundation

**Capstone:** Portfolio case study #1 | **Level:** L2

| Week | Topics | Project |
|------|--------|---------|
| W1 | Design fundamentals + Figma mastery | UI kit creation |
| W2 | User research, personas, journey maps | Research report |
| W3 | Wireframing, prototyping, interaction design | App prototype (mid-fi) |
| W4 | High-fi design, design system, case study | **Portfolio case study #1** |

**W4 acceptance criteria:**
- [ ] Problem statement, research summary, solution, outcomes
- [ ] Before/after or iteration screenshots
- [ ] Behance/Dribbble-ready presentation (5+ slides)

## 3-Month Job Ready

**Capstone:** End-to-end UX case study with live HTML/CSS build | **Level:** L3

| Month | Topics | Projects |
|-------|--------|----------|
| M1 | Visual design, Figma, responsive web design | Landing page + mobile UI |
| M2 | UX research, IA, wireframes, usability testing | 2 app redesigns |
| M3 | Design systems, accessibility, Tailwind implementation, capstone | **End-to-end UX case study + HTML/CSS build** |

**M3 Capstone acceptance criteria:**
- [ ] 5+ user research participants (or heuristic evaluation)
- [ ] WCAG AA contrast compliance documented
- [ ] Responsive HTML/CSS implementation matches Figma
- [ ] Usability test findings incorporated in final design

## Appendix: Mentor Notes

**Common pitfalls:** Skipping research; inconsistent spacing; poor contrast ratios; designs that ignore developer constraints.

**Hardware:** Any laptop; drawing tablet optional; second monitor recommended for Figma.
