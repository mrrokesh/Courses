## Duration Tiers

| Duration | Hours | Depth | Projects | Outcome Level |
|----------|-------|-------|----------|---------------|
| **2 Weeks** (Crash) | 40–50 hrs | Awareness + core syntax | 1 mini project | Can build basic apps; needs mentorship |
| **1 Month** | 80–100 hrs | Foundation + workflow | 2 guided projects | Junior-ready basics |
| **3 Months** | 250–300 hrs | Intermediate + integration | 3–4 projects | Internship-ready |
| **6 Months** | 500–600 hrs | Advanced + deployment | 5–6 + 1 capstone | Job-ready (entry–mid) |
| **9 Months** (Advanced) | 750–900 hrs | Expert + specialization | 8–10 + 2 capstones | Industry-ready + certification prep |

### Outcome Details by Tier

**2 Weeks — Crash**
- Understand core concepts and syntax
- Complete one guided mini project with mentor support
- Deploy or demo a basic working application
- Needs continued mentorship for production work

**1 Month — Foundation**
- Solid grasp of fundamentals and developer workflow
- Use Git/GitHub for version control
- Build 2 guided projects independently with review
- Ready for internship screening at junior level

**3 Months — Job Ready**
- Integrate frontend, backend, and database concepts
- Complete 3–4 projects including one capstone
- Write clean, documented code with README portfolio entries
- Eligible for internship and entry-level roles

**6 Months — Professional**
- Build production-grade applications with auth, deployment, and testing
- Complete 5–6 projects plus one major capstone
- Participate in code reviews and mock interviews
- Job-ready for entry to mid-level positions

**9 Months — Advanced**
- Master specialization topics (TypeScript, microservices, MLOps, etc.)
- Complete 8–10 projects plus two capstones
- Prepare for industry certifications
- Industry-ready with system design and portfolio depth

## Weekly Structure (All Durations)

| Day | Activity | Focus |
|-----|----------|-------|
| **Mon–Thu** | Concept + live coding lab | 70% hands-on; mentor demonstrates then students build |
| **Fri** | Mini-project / assignment submission | Apply week's topics to a scoped deliverable |
| **Sat** | Code review + doubt clearing | Peer review, mentor feedback, Q&A session |
| **Sun** | Revision + documentation | Study notes, GitHub README, portfolio update |

### Weekly Hour Allocation (Example: 3-Month Track)

| Activity | Hours/Week | % |
|----------|------------|---|
| Live labs (Mon–Thu) | 12–16 hrs | 70% |
| Theory / concept sessions | 4–6 hrs | 30% |
| Project work (Fri) | 3–4 hrs | — |
| Review + docs (Sat–Sun) | 2–3 hrs | — |
| **Total** | **~20–25 hrs/week** | — |

## Assessment Formula

| Component | Weight | Criteria |
|-----------|--------|----------|
| **Weekly Quiz** | 20% | Concept retention; 10–15 MCQ + 1 short answer |
| **Lab Completion** | 40% | All lab checkpoints submitted; code runs; follows rubric |
| **Project** | 30% | Functionality, code quality, demo, documentation |
| **Portfolio** | 10% | GitHub README, screenshots, live demo link, commit history |

### Example Grade Calculation

| Component | Score | Weight | Weighted |
|-----------|-------|--------|----------|
| Quiz average | 85% | 20% | 17.0 |
| Lab completion | 90% | 40% | 36.0 |
| Project | 78% | 30% | 23.4 |
| Portfolio | 95% | 10% | 9.5 |
| **Final Grade** | | | **85.9%** |

**Pass threshold:** 60% overall with minimum 50% on project component.

## GitHub Portfolio Requirements

Every student maintains a public GitHub profile with:

1. **Profile README** — Name, track, skills, contact
2. **One repo per project** with structured README containing:
   - Project title and one-line description
   - Tech stack used
   - Screenshots or demo GIF
   - Setup/installation instructions
   - Live demo URL (when applicable)
   - What you learned (3–5 bullets)

### README Template Snippet

```markdown
# Project Name

One-line description of what this project does.

## Tech Stack
- React, Node.js, MongoDB, etc.

## Features
- Feature 1
- Feature 2

## Setup
1. Clone repo
2. npm install
3. npm start

## Live Demo
https://your-app.vercel.app

## What I Learned
- Bullet 1
- Bullet 2
```
